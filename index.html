What is pattern causality ?

Pattern causality(PC) is a novel causal eect analysis tool based on chaos theory and systematics, construct the state space by embedding the original sequence into phase space and explore its hidden patterns to estimate the causality.

Papers

The introduction of algorithm can be found in

  1. Stavroglou, Pantelous, Stanley and Zuev (2019)

Usage in R

First we need to import the package into R environment.

library(patterncausality)

This package contains several datasets like climate indices and stock price, we can import the climate dataset first.

data(climate_indices)

The series will look like this which is obviously nonlinear sequences here.

With the parameter $E=3$ and $\tau=1$ we can construct the state space (attractor) and plot it. This plot function is based on plot3D package and the plot parameters will be the same as this package.

state <- stateSpace(climate_indices$AO, 3,1)
plot(state)

As the basic usage, it can be calculated by pcLightweight function directly, we can just extract the data and put it into this function and it will do the whole process of PC and output the result of pattern causality.

X <- climate_indices$AO
Y <- climate_indices$AAO
pc <- pcLightweight(X, Y, E = 3, tau = 1, metric = "euclidean", h = 1, 
                    weighted = TRUE, verbose = FALSE)
print(pc)

The parameters $E$ and $\tau$ are the same as state space construction part and metric is the parameter for distance type, h means predicting h steps ahead on manifold and weighted uses the erf function here to calculate the PC strength.

For this pc_fit class we can also be plotted one by one.

plot(pc)

As for the climate system, we can use the different pairs of parameter $E$ and $\tau$ and calculate its strength in the system.

dataset <- climate_indices[, -1] # remove the date column
parameter <- optimalParametersSearch(Emax = 3, tauMax = 3, 
                                     metric = "euclidean", dataset = dataset)
print(parameter)

Further more, we can also find the PC strength in each time point, this process can be done by pcFullDetails function and we can also plot it directly.

X <- climate_indices$AO
Y <- climate_indices$AAO
detail <- pcFullDetails(X, Y, E = 3, tau = 1, metric = "euclidean", h = 1, 
                        weighted = TRUE, verbose = FALSE)
plot_causality(detail, type="total")

Cross-Validation Analysis

As for the PC strength in different length, we can use pcCrossValidation function to estimate the strength with different length, the random parameter here controls the resampled sequence is ordered or random.

X <- climate_indices$AO
Y <- climate_indices$AAO
result <- pcCrossValidation(
  X = X, 
  Y = Y,  
  numberset = seq(30, length(X), by = 5),
  E = 3,
  tau = 1,
  metric = "euclidean",
  h = 1,
  weighted = TRUE,
  random = FALSE
)

It can also be visualized by plot function directly.

We can also find its convergence here.

plot(result)

Otherwise, we also provide a method to boostrap the sample and obtain the strength with bootstrap method to estimate its confidence interval. We can also find that the interval is smaller and smaller with the length increasing.

set.seed(123)
X <- climate_indices$AO
Y <- climate_indices$AAO
result_boot <- pcCrossValidation(
  X = X,
  Y = Y,
  numberset = seq(30, length(X), by = 10),
  E = 3,
  tau = 1,
  metric = "euclidean",
  h = 1,
  weighted = TRUE,
  random = TRUE,
  bootstrap = 20
)

plot(result_boot)

Matrix Analysis

For the high dimension dataset, we can calcuatel all the pairs in the dataset directly with pcMatrix function. In this example we will use DJS dataset which contains 29 sequences.

data(DJS)

Then we can calculate all the pairs in this dataset directly and the result will be save in pc_matrix class.

dataset <- DJS[,-1]
result <- pcMatrix(
  dataset = dataset, 
  E = 3,           # Embedding dimension
  tau = 1,         # Time delay
  metric = "euclidean",
  h = 1,           # Prediction horizon
  weighted = FALSE  # Unweighted analysis
)

Parallel Computing

As some examples, the parallel parameter is included in some existing functions which gives possibilities to calculate it fater when bootstrap number is really large, we can set the parameters directly like

set.seed(123)
X <- climate_indices$AO
Y <- climate_indices$AAO
result_boot <- pcCrossValidation(
  X = X,
  Y = Y,
  numberset = seq(30, length(X), by = 10),
  E = 3,
  tau = 1,
  metric = "euclidean",
  h = 1,
  weighted = TRUE,
  random = TRUE,
  bootstrap = 100,
  n_cores = 4, # it can be set by parallel::detectCores()
)
